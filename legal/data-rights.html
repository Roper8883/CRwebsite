Use the in-app delete and account controls where available

Current product flows indicate that an in-app Delete Account & Data path exists in Settings on supported builds.

If the in-app path is available, that is the cleanest way to start an account-deletion flow for that build.

Requests you can make

  • Account deletion
  • Data-access request
  • Correction request
  • Export request
  • Restriction, objection, opt-out, or portability request where applicable law provides that right
  • Privacy enquiry or complaint

What to include in a request

  • The app name you are using
  • Your email address or account identifier
  • The type of request you are making
  • Any context needed to identify the relevant account or record
  • Screenshots if they help explain the issue

What happens next

Current backend privacy-ops routes show that export and delete requests can be queued and tracked.

A delete request may not mean immediate erasure if some records still need to be retained for lawful security, support, fraud-prevention, or recordkeeping reasons.

We may need to verify that a request relates to your account before taking action. If a request is denied or limited, we will aim to explain the practical reason and the next support route available.

Regional rights context

Australia: requests may include access or correction of personal information, and privacy complaints can be routed through support before any external complaint route.

United Kingdom: requests may include access, rectification, erasure, restriction, objection, portability, and complaint support where UK GDPR-style rights apply.

United States: requests may include access/know, delete, correct, opt-out, limit-use, and non-discrimination enquiries where state privacy laws apply, including California law where applicable.

Where to send the request

  • CapstoneReady: capstoneready@outlook.com
  • AM2Ready: am2ready@outlook.com
  • JourneymanReady: journeymanready@outlook.com
