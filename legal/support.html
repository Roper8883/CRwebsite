What to include

Make the first support message more useful

  • App name and region
  • Device and operating system
  • App version/build if visible
  • A short description of what happened
  • Screenshots if they help
  • Whether the issue happened in the app, on the public website, or on the secure sign-in host
Privacy and deletion

Use the same support lane for data-rights help when needed

  • Account deletion support
  • Privacy request routing
  • Data access or correction queries
  • Entitlement or restore issues that need manual review
Need privacy or deletion help? Use the dedicated Data Rights page for the in-app delete path, request guidance, and what to include in an export, correction, or deletion request.
